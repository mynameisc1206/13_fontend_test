# 13_통합구현_천웅재

### 1. 외부 및 내부 시스템의 정의와 역할
외부 시스템은 학생의 인적 사항과 출석 상태를 배열 형태로 보유하고 있는 server-data.js이며 
현재 프로젝트에서 데이터의 원천 역할을 수행합니다. 반면 내부 시스템은 script 태그를 통해 
외부로부터 전달받은 데이터(server-data.js)를 브라우저에 구조화하여 배치하는 학생출석관리.html과 Css(style태그) 이며,
이 파일들은 데이터를 동적으로 제어하는 내부 로직이 결합된 환경을 보여줍니다.


### 2. 연계 및 화면 출력 데이터 항목 정리  
외부 서버 데이터 중 시스템 내부에서 각 객체를 고유하게 식별하기 위한 id 값을 제외하고, 
학생 성명(name), 학급 정보(className), 출석 상태(status)가 핵심 연계 데이터로 판단되었습니다. 
이 데이터들은 서버에서 제공하는 원천 정보 중 사용자에게 학생들을 구분 할 수 있는 정보를
제공하기 위한 목적으로 선별되었으며, 화면상에서 학생 리스트를 구성하고 
상태별 필터링 기능을 구현하는 데 필수적인 요소로 사용됩니다.


### 3. 데이터 연계 및 화면 반영 흐름 
데이터 연계 흐름은 외부 Javascript 파일에 저장된 데이터를 내부 시스템으로 로드하는 것에서 시작되며, 
사용자가 버튼을 클릭하는 순간 '클릭 이벤트(onclick="fliterStudents")'가 작동하여 필요한 데이터를 선별하는 과정을 거칩니다. 
선별된 데이터는 자바스크립트의 반복문(forEach)을 통해 가공되며, 최종적으로 DOM(문서 객체 모델) 조작을 통해
HTML 요소로 생성되어 브라우저 화면에 실시간으로 렌더링되는 동적 업데이트 방식을 따릅니다.


### 4. 본 과제가 데이터 연계 구현 과제인 이유 
본 과제는 정적인 텍스트를 보여주는 것에 그치지 않고, 서로 다른 환경(외부 파일과 웹 브라우저)에 존재하는 시스템 간의 인터페이스
를 구축하는 과정을 포함하고 있기 때문입니다. 외부의 원천 데이터를 손실 없이 수신하고, 이를 내부 시스템의 조건(onclick="fliterStudents")에 맞춰 가공하여
최종적으로 사용자 화면에 일치시키는 전 과정을 다루고 있다는 점에서 전형적인 프론트엔드 데이터 연계 및 통합 구현의 핵심 모델이라 할 수 있습니다.
